# AI Orchestrator 사용자 가이드

문서 처리·태깅·검색 프로그램의 모든 버튼과 기능을 설명합니다.

---

## 1. 메인 화면 (app.py)

### 1.1 설정 영역

| 항목 | 설명 |
|------|------|
| **입력 폴더** | 문서·이미지가 있는 폴더 경로. DOCX, PPTX, PDF, TXT, MD, HWP, XLS/XLSX 등 지원 |
| **출력 폴더** | 처리 결과가 저장되는 폴더 |
| **변환 임시폴더** | PPT 등 변환 시 사용하는 임시 폴더 |
| **soffice.exe** | LibreOffice/OpenOffice 경로. PPT→이미지 변환에 필요 |
| **inference_scope** | public / internal / restricted 중 선택 |
| **문서 처리 시 주제문장 입력 요청** | 체크 시 각 문서 처리 전에 주제문장 입력 창 표시 |
| **의미 보정 사용** | 태깅 시 의미 기반 신뢰도 보정 사용 여부 |
| **OCR 활성화** | 문서 내 이미지 OCR 처리 여부 |
| **최대 OCR 이미지** | 문서당 OCR 처리할 최대 이미지 수 (0~50) |

**찾기 버튼** (입력/출력/임시폴더): 폴더 선택 대화상자 열기  
**파일 선택** (soffice): soffice.exe 파일 선택

---

### 1.2 문서 처리 버튼

| 버튼 | 동작 |
|------|------|
| **문서/폴더 일괄 처리 시작(누적)** | 입력 폴더를 스캔하여 문서·이미지를 처리. 기존 데이터 유지, 중복은 스킵. 처리 중에는 버튼 비활성화 |
| **중지** | 현재 파일 처리 후 일괄 처리를 중단. 처리 중에만 활성화 |
| **순수 이미지 추가(폴더/파일)** | 문서와 무관한 이미지를 라벨링 큐에 추가. 폴더 또는 파일 선택 가능 |
| **이미지 라벨링(범용 5문항)** | images_queue.jsonl에 있는 이미지에 대해 라벨링 UI 실행. 큐에 이미지가 있을 때만 활성화 |
| **업무 레코드/보고서 생성(수동 입력)** | 수동 입력 기반 레코드·보고서 작성 창 열기 |

---

### 1.3 청킹·검색·분석 버튼

| 버튼 | 동작 |
|------|------|
| **정제 메타DOCX 생성(전체)** | output/metadata_docs 폴더에 문서별 정제 DOCX와 _IMAGE_DICTIONARY.docx 생성 |
| **문서 청킹 처리(선택)** | 문서 선택 → 청킹 설정 → 선택한 문서에 대해 청킹 실행 (chunks.json, analysis_report.json 생성) |
| **청킹 결과 보기** | 문서 선택 후 분석 뷰어에서 청킹 결과 확인 |
| **문서 검색** | 키워드·태그·장르 기반 문서 검색 창 열기 |
| **태깅 결과 보기** | 문서별 태깅 결과 목록·상세 보기 |
| **문서 분류 대시보드** | 문서 분류 현황·카테고리 관리·K-means 자동 분류 |

---

## 2. 문서 선택 창 (DocumentSelectWindow)

라벨링·청킹·분석 시 문서를 고를 때 사용합니다.

| 버튼 | 동작 |
|------|------|
| **문서함 갱신** | 출력 폴더 기준으로 문서 목록 다시 로드 |
| **입력폴더 재동기화** | 입력 폴더와 비교해 없는 문서 정리, payload·큐 정리 |
| **선택** | 리스트에서 선택한 문서를 확정하고 창 닫기 |

---

## 3. 주제문장 입력 창 (DocumentTopicWindow)

문서 처리 시 주제문장 입력 모드일 때 표시됩니다.

| 버튼 | 동작 |
|------|------|
| **건너뛰기 (오토태깅 사용)** | 주제문장 없이 기존 오토태깅으로 진행 |
| **확인** | 입력한 주제문장을 저장하고 문서 처리 진행 |

---

## 4. 청킹 설정 창 (ChunkingSettingsWindow)

| 버튼 | 동작 |
|------|------|
| **취소** | 설정을 적용하지 않고 창 닫기 |
| **확인** | target_chars, min_chars, overlap_chars를 적용하고 청킹 실행 |

---

## 5. 이미지 라벨링 창 (ImageReviewWindow)

이미지에 제목·고객사·분류·설명·태그를 입력합니다.

| 버튼 | 동작 |
|------|------|
| **이전** | 이전 이미지로 이동 |
| **다음** | 다음 이미지로 이동 |
| **마지막 이미지로** | 큐의 마지막 이미지로 이동 |
| **저장** | 현재 입력 내용을 저장하고 다음 이미지로 이동 |
| **저장하지 않음(무시)** | 현재 이미지를 무시 처리하고 다음으로 이동 |
| **완료하기** | 라벨링 창 닫기. 미처리 이미지가 있으면 확인 메시지 표시 |
| **큐 새로고침** | images_queue.jsonl을 다시 읽어 목록 갱신 |

---

## 6. 업무 레코드/보고서 생성 (RecordBuilderWindow)

| 버튼 | 동작 |
|------|------|
| **텍스트 블록 추가** | 텍스트만 있는 블록 추가 |
| **사진+텍스트 블록 추가** | 사진과 텍스트가 있는 블록 추가 (폴더/파일 선택) |
| **선택 블록 삭제** | 선택한 블록 삭제 |
| **저장(초안)** | record.json과 첨부 파일만 저장 |
| **저장 + 출력(보고서+RAG+태깅)** | 저장 후 report.md, chunks.jsonl, RAG export 생성 및 태깅 실행 |
| **출력(보고서+RAG+태깅)** | 저장된 초안 기준으로 보고서·RAG·태깅만 실행 |

---

## 7. 문서 검색 창 (DocumentSearchWindow)

| 버튼 | 동작 |
|------|------|
| **검색** | 검색어·장르·태그 필터로 문서 검색 |
| **인덱스 재구축** | 검색 인덱스를 다시 생성 (문서 추가·수정 후 사용) |

**체크박스**
- **주제문장이 있는 문서만 검색**: 주제문장이 있는 문서만 대상
- **최적화 검색 사용 (2단계 검색)**: 2단계 검색 알고리즘 사용

**검색 결과 피드백 창**
- **👍 좋아요** / **👎 좋아요 안함**: 검색 결과 피드백 저장
- **상세 보기**: 문서 상세 내용 보기
- **취소**: 피드백 창 닫기

---

## 8. 태깅 결과 보기 (TaggingViewer)

| 버튼 | 동작 |
|------|------|
| **새로고침** | 태깅 데이터 다시 로드 |
| **상세 보기** | 선택한 문서의 태깅 상세 보기 |
| **닫기** | 창 닫기 |

---

## 9. 문서 분류 대시보드 (ClassificationDashboard)

| 버튼 | 동작 |
|------|------|
| **카테고리 추가** | 새 카테고리 추가 |
| **인덱스 갱신** | 문서 인덱스 재구축 |
| **K-means 자동 분류** | K-means로 문서 자동 분류 |

**트리뷰 우클릭 메뉴**
- **카테고리 변경**: 선택 문서의 카테고리 변경
- **문서 상세 보기**: 분석 뷰어에서 문서 상세 보기

## 11. 실행 방법

```powershell

.\.venv\Scripts\Activate.ps1

# 앱 실행 (프로젝트 루트에서)
python meta\app.py
```

## 12. 출력 구조

```
output/
├── workspace_payload.jsonl    # 처리된 문서 메타데이터
├── images_queue.jsonl         # 라벨링 대상 이미지 큐
├── images_labels.jsonl        # 라벨 저장 이력
├── failed_docs.jsonl          # 처리 실패 문서
├── metadata_docs/             # 정제 DOCX
│   ├── <문서명>.docx
│   └── _IMAGE_DICTIONARY.docx
├── core/
│   └── chunks.jsonl           # 청킹 결과 (전체)
└── <doc_id>/                  # 문서별 폴더
    ├── document_facts.json
    ├── extracted.json
    ├── chunks.json
    ├── analysis_report.json
    └── images_metadata.json
```
